# Contributing

Thanks for your interest in contributing to WP Boilerplate!

## Getting started

1. Fork the repository
2. Clone your fork and run `npm run setup`
3. Create a feature branch: `git checkout -b my-feature`
4. Make your changes
5. Test the full setup flow: `npm run reset && npm run setup`
6. Commit and push to your fork
7. Open a pull request

## Guidelines

- Keep the boilerplate minimal — avoid adding opinionated features that not everyone needs
- Test with at least two package managers (npm + one other)
- Make sure `npm run build` produces a working theme
- Update the README if your change affects the setup flow or project structure

## Reporting bugs

Open an issue with:
- Your OS and Node.js version
- The package manager you used
- Steps to reproduce
- The error output

## Questions?

Open an issue — happy to help.
